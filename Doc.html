Documentación del Proyecto: Gestión de Asistencia y Asignación de Código

Descripción del Proyecto

El sistema de gestión de asistencia permite a los usuarios agregar nuevos alumnos y ver la asistencia de los mismos por grupo. Este sistema se ha desarrollado utilizando HTML, CSS, JavaScript y PHP, y utiliza Bootstrap para la interfaz de usuario. También se ha añadido la funcionalidad de búsqueda por código de alumno, que permite encontrar rápidamente la información de un estudiante específico. Además, se han implementado websockets para mejorar la comunicación en tiempo real entre el servidor y los clientes.

Requisitos

  • Un servidor web que soporte PHP (como Apache o Nginx).
  • Un entorno de desarrollo para pruebas (por ejemplo, XAMPP, WAMP o MAMP).
  • Conexión a una base de datos (MySQL o similar) para almacenar información de alumnos y asistencia.
  • Extensión de PHP para websockets (como Ratchet o Swoole).

Instalación

  1. Descargar y configurar el servidor:

    Asegúrate de tener un servidor web instalado y en funcionamiento. Coloca el proyecto en la carpeta de documentos del servidor (por ejemplo, htdocs en XAMPP).

  2. Crear base de datos:

    Crea una base de datos en MySQL para almacenar la información de los alumnos. Crea una tabla para los alumnos con los siguientes campos:

    - matricula
- nombre_completo
- carrera
- grupo
- codigo (opcional)
- asistencia (booleano para indicar presencia)
  3. Configurar conexión a la base de datos:

    En index.php, configura los parámetros de conexión a tu base de datos.

  4. Configurar websockets:

    Asegúrate de tener la extensión de websockets instalada y configurada correctamente en tu entorno. Esto permitirá la comunicación en tiempo real.

Funcionalidades

Agregar Alumnos Nuevos

La página agregar_alumno.html permite a los usuarios ingresar la matrícula, nombre completo, carrera, grupo y un código de barras opcional para cada nuevo alumno. Al enviar el formulario, se realiza una solicitud AJAX a index.php para almacenar la información en la base de datos.

Ver Asistencia por Grupo

La página ver_asistencia.html permite a los usuarios ingresar un grupo y visualizar la lista de alumnos presentes en ese grupo. Se realiza una solicitud AJAX para obtener la lista de estudiantes y sus estados de asistencia.

Generación de PDF

Desde ver_asistencia.html, los usuarios pueden descargar un PDF con la lista de asistencia generada a partir de la tabla visible en la página.

Buscar por Código de Alumno

La página buscar_por_codigo.html permite a los usuarios ingresar un código de alumno para buscar su información específica. Se realiza una solicitud AJAX a index.php y muestra el nombre completo, carrera y logros obtenidos del alumno correspondiente.

Presentador

En la página presentador.html, se ha implementado websockets para permitir la comunicación en tiempo real sobre la asistencia de los alumnos. Esto mejora la interactividad y la respuesta inmediata al estado de asistencia.

Cambios Realizados en las Páginas

  • Se implementaron websockets en presentador.html para permitir actualizaciones en tiempo real sobre la asistencia.
  • Se mejoró la interfaz de usuario de todas las páginas para hacerla más intuitiva y accesible.
  • Se optimizaron las solicitudes AJAX para mejorar el rendimiento y la velocidad de respuesta.

Mantenimiento

Asegúrate de mantener la base de datos actualizada con los nuevos alumnos y sus asistencias. Revisa periódicamente el código para realizar mejoras y asegurar su correcto funcionamiento. Además, verifica el funcionamiento de los websockets para garantizar una comunicación fluida.

Posibles Problemas y Soluciones

Problemas de Conexión a la Base de Datos

Si encuentras errores al intentar conectar a la base de datos, asegúrate de que:

  • El servidor de base de datos está en funcionamiento.
  • Las credenciales de la base de datos en index.php son correctas.
  • La base de datos y las tablas necesarias han sido creadas.

Problemas con Websockets

Si los websockets no funcionan como se espera, verifica lo siguiente:

  • La extensión de websockets está instalada y habilitada en tu servidor PHP.
  • El servidor de websockets está en funcionamiento y escuchando en el puerto correcto.
  • No hay bloqueos de firewall que impidan las conexiones a los websockets.

Problemas con las Solicitudes AJAX

Si las solicitudes AJAX no funcionan:

  • Verifica la consola del navegador para ver si hay errores de JavaScript.
  • Asegúrate de que las rutas a los archivos PHP sean correctas.
  • Revisa el código PHP para detectar errores de sintaxis o lógica.
